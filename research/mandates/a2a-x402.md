## What

a2a-x402 is a Google extension that brings x402 on-chain payment semantics to the Agent2Agent (A2A) protocol. It enables agents to monetize their services within A2A communication flows using x402 payment settlement.

## Who

Developer: **Google** (google-agentic-commerce). Companion to AP2 and part of the Google agentic commerce suite.

## Mechanism

a2a-x402 wraps x402 payment flows within A2A agent message envelopes:

1. Agent A calls Agent B via A2A protocol
2. Agent B responds with A2A message containing x402 payment requirement
3. Agent A resolves payment via x402 facilitator
4. Agent A replays A2A call with payment receipt token
5. Agent B verifies receipt and returns the service

## When to choose

- You're building agent-to-agent services within A2A ecosystems
- You want x402 on-chain settlement for A2A monetization
- You need to charge other agents for your agent's services
