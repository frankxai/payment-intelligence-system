# Research — x402 Protocol

## What

x402 is an HTTP-native payment protocol that uses the `402 Payment Required` HTTP status code to gate access to APIs, content, or services behind a payment. An agent receives a 402 response with a payment payload header, submits the payment to a facilitator, receives a receipt token, and replays the request with the token to get access.

## Who

Steward: **x402 Foundation** (independent nonprofit). Founding members (as of 2026-06): Coinbase, Visa, Mastercard, Stripe, Google, Cloudflare. Coinbase ships the reference facilitator and x402-js SDK. Primary development language: TypeScript, with Python and Go clients.

## Mechanism

1. Agent calls `GET /resource` or `POST /api/v1/action`
2. Server responds `402 Payment Required` with header `X-Payment-Required: {amount, currency, facilitator, scheme}`
3. Agent submits payment to facilitator (e.g., `https://x402.coinbase.com/facilitate`) with `{payload, amount}`
4. Facilitator verifies balance, deducts payment, returns signed receipt token
5. Agent replays request with header `X-Payment-Token: <receipt>`
6. Server verifies receipt with facilitator (or offline using facilitator public key) and returns resource

**Settlement:** On-chain via Base L2 (Coinbase default facilitator). Other facilitators may use different chains. Authorization latency: typically <500ms. Settlement finality: 12-18 seconds on Base (as of 2026-06).

**Payment amounts:** Typically denominated in USDC. Minimum amounts as low as $0.0001; practical floor set by gas overhead.

## When to choose

- API endpoint monetization for AI agent callers (the primary use case)
- Sub-$50 per-call payments where on-chain settlement is acceptable
- Trustless settlement without PSP relationship (no Stripe/PayPal account required for payer)
- Global counterparties without card accounts
- When you want composability with other x402 Foundation member networks

**Not for:** Retail consumer checkout (use ACP/UCP), mandates requiring Byzantine consensus (use AP2), fiat-only settlement (use Stripe MPP), sub-cent with gas-sensitive constraints (use L402).

## Verified facts as of 2026-06

- **~165M agent transactions** processed via x402 as of 2026-06 (x402 Foundation)
- **x402 Foundation members:** Coinbase, Visa, Mastercard, Stripe, Google, Cloudflare (founding)
- **Primary facilitator:** Coinbase (`x402-facilitator` open-source on GitHub)
- **Chain:** Base L2 (Coinbase), with multi-chain facilitator support planned
- **SDKs:** `x402-js` (TypeScript/Node), Python client, Go client (community)
- **MCP integration:** `x402-mcp-server` enables payment-gating of MCP tools directly
