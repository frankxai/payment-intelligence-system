## What

L402 is an HTTP 402 payment protocol built on the Bitcoin Lightning Network. It pairs a Lightning invoice with a macaroon bearer credential: the client pays the invoice and receives a macaroon that gates access to the resource. L402 is the original revival of `402 Payment Required` that preceded x402.

## Who

Maintainer: **Lightning Labs** (lightninglabs). The same team that ships LND (Lightning Network Daemon). L402 is part of the Lightning Agent Tools suite.

## Mechanism

1. Agent calls resource endpoint
2. Server returns `402 Payment Required` with a Lightning invoice (BOLT11) and an empty macaroon
3. Agent pays the Lightning invoice via LND or LNC (Lightning Node Connect)
4. Payment proof (preimage) is received
5. Agent presents the macaroon + preimage to unlock the resource
6. Server verifies the preimage matches the invoice hash; returns resource and grants macaroon

**Settlement:** Lightning Network channel settlement. Near-instant (seconds) when Lightning channels are funded. On-chain settlement when channels close. Payments denominated in satoshis (BTC).

**Macaroon credentials:** Macaroons are bearer tokens with caveats (time limits, IP restrictions, resource scoping). The agent can present the macaroon for repeat access without re-paying until the macaroon caveat conditions are violated.

## When to choose

- Sub-cent micropayments where gas costs would make x402 impractical
- Counterparty already runs Lightning node infrastructure (exchanges, wallets, node operators)
- You want credential-gated access (macaroon caveats) alongside payment
- Streaming/pay-as-you-go access to data feeds with existing Lightning integration

**Not for:** Agents without Lightning wallet infrastructure, merchants needing fiat settlement, retail consumer checkout.
